User Profile

Technical Description

  • HTML Techniques

    • The userprofile.html file uses a combination of semantic HTML5 tags to create a structured and accessible user profile page. Key tags include:

    • header and nav: These tags are used to define the navigation bar at the top of the page. The nav tag is particularly useful for defining a set of navigation links, enhancing both functionality and accessibility.
    • Section: This tag is used multiple times to divide the page into logical sections such as user details, work skills, volunteering calendar, and contact information. This helps in organizing content and improving readability.
    • div: Used for creating containers and grouping elements together. For example, the userProfile, userDetails, workSkills, and timeLineAbout sections are wrapped in div tags to manage their layout and styling.
    • img: Used for displaying images, such as the user's profile picture and background image. The alt attribute is included for accessibility, providing a text description for screen readers.
    • form: Used for the contact form, allowing users to send messages. The form includes input fields for name, email, and message, with appropriate labels and placeholders for user guidance.
  • CSS Techniques
    • Flexbox and Grid: These CSS layout modules are used to create responsive and flexible layouts. For example, the main container uses flexbox to align sections vertically, while the calendar and tabs use grid layouts for better control over spacing and alignment.
    • Media Queries: Used extensively to ensure the page is responsive and adapts to different screen sizes. This includes adjusting font sizes, widths, and positions based on the viewport size.
    • Custom Properties (CSS Variables): Used for defining color schemes and other reusable styles, making it easier to maintain and update the design.

    Accessibility

    • Semantic HTML: The use of semantic HTML tags like header, nav, section, and form helps screen readers and other assistive technologies understand the structure and purpose of the content.
    • Alt Text for Images: All images include alt attributes with descriptive text, ensuring that users who rely on screen readers can understand the content of the images.
    • Color Contrast: The color scheme ensures sufficient contrast between text and background colors, making the content readable for users with visual impairments.
    • Responsive Design: The use of media queries ensures that the page is usable on a variety of devices and screen sizes, including mobile phones and tablets.

    Challenges and Lessons Learned

    During the implementation of the sitemap.html, userprofile.html, and content_ST04.html pages, several challenges were encountered that provided valuable learning experiences.

    One significant challenge was ensuring the responsiveness of the site map visualization in sitemap.html. The complex SVG structure required precise positioning and scaling to maintain clarity across different devices. This was resolved by extensively using CSS media queries and flexible SVG properties, allowing the site map to adapt seamlessly to various screen sizes. The lesson learned was the importance of thorough testing and iterative adjustments to achieve a responsive design that works well on all devices.

    For the userprofile.html page, managing the interactive tabbed interface and form validation posed initial difficulties. The JavaScript logic for switching between tabs and validating form inputs had to be carefully implemented to ensure a smooth user experience. By breaking down the functionality into smaller, manageable functions and thoroughly testing each part, these challenges were overcome. This reinforced the value of modular coding and rigorous testing to ensure robust functionality.

    In content_ST04.html, the challenge was to create a visually appealing and accessible layout for a large amount of content, including events, publications, and news. Using CSS Grid and Flexbox, a flexible and responsive layout was achieved. The lesson here was the effectiveness of modern CSS layout techniques in managing complex content structures while maintaining accessibility and usability.

    Overall, these challenges highlighted the importance of patience, attention to detail, and continuous learning in web development. Each obstacle provided an opportunity to deepen understanding of HTML, CSS, and JavaScript, ultimately leading to more resilient and user-friendly web pages.

    Compliance

    The web pages created adhere to the regulations governing web page publication as outlined by Janet. These regulations emphasize the importance of accessibility, security, and user privacy. To ensure compliance, the following measures were taken

    1. Accessibility: The pages were designed with accessibility in mind, following the Web Content Accessibility Guidelines (WCAG). This includes the use of semantic HTML tags, alt text for images, and proper form labeling. Additionally, the pages were tested for color contrast and keyboard navigability to ensure they are usable by individuals with disabilities.
    2. Security: To protect user data and ensure secure communication, the pages were designed to be served over HTTPS. This encrypts data transmitted between the user and the server, safeguarding sensitive information. Furthermore, the forms on the pages were designed with input validation to prevent common security vulnerabilities such as SQL injection and cross-site scripting (XSS).
    3. User Privacy: The pages do not collect or store personal information without explicit user consent. Any data collected through forms is handled in accordance with privacy laws and regulations. Users are informed about the data being collected and how it will be used, ensuring transparency and compliance with privacy standards.

    By adhering to these regulations, the web pages ensure a safe, accessible, and user-friendly experience for all visitors.
