Splash Page

Technical Description

The splash screen is designed using HTML and CSS

HTML techniques

  • The document uses (!DOCTYPE html) for HTML5 standards.
  • CSS files are linked using (link rel="stylesheet") for global and splash-specific styles.
  • (h1) contains the main title, with a text animation effect for emphasis.
  • (p) under (div class="mission") describes the mission clearly and concisely.
  • (h3) and (ul) list the group members in an organized way.

CSS techniques

  • The (.splash) section sets a full-page background image.
  • (background-size: cover;) ensures the image adapts to different screen sizes.
  • (text-align: center;) aligns text for better readability.
  • (font-size) and (font-weight) create a structured hierarchy in headings and body text.
  • (.splash_inner) uses (display: flex; flex-direction: column; align-items: center; justify-content: center;) to center all content vertically and horizontally.
  • .text-focus-in and .text-focus-in-mission use CSS keyframe animations to create a fade-in effect for a smooth user experience.

Accessibility

These techniques below improve the accessibility of the splash screen.

  • High contrast text - White text on a dark overlay improves readability.
  • Structured content - HTML elements (h1,p,ul) are used for structured content.
  • Flexible layout - The use of max-width and text-align: center; ensures text remains legible on different screen sizes.
  • Readable timing - 4 seconds delay before rederecting to home page gives time to read the content.
  • Clear typography - Font is clear and legible (font-family: )
  • Flexible layout with vh units - Ensures the splash screen fits all screen sizes, including mobile devices.

Volunteer Page

Technical Description

The Volunteer Page is implemented using HTML,CSS and JS to provide a structured and visually appealing interface.

HTML elements

  • (header and nav) - Contains a logo (img), a navigation menu (ul), and a responsive hamburger menu (div class="hamburger").
  • (section class="volunteer-section") - Displays volunteer programs with a header (h2), sorting button (button class="best-match-btn"), and program cards (div class="program-container")
  • (div class="program-card") - Each card includes an image (img), a title (h3), description buttons (button class="info-btn">), and action buttons (button class="apply-btn")
  • (section id="reviews") - Displays user reviews with ratings, names, and dates.

CSS techniques

  • Global Styling - Ensures consistent spacing and typography across the page.
  • Navigation Bar - (Uses display: flex;) for proper alignment.
  • Dropdown Menu - Becomes visible on interaction (display: none; initially).
  • Footer - Uses (display: flex;) ,j(ustify-content: space-between;) for equal spacing.

Accessibility

These techniques below improve the accessibility of the volunteer page.

  • Semantic HTML for better screen reader navigation.
  • Keyboard Navigation for dropdown menus and buttons.
  • Alt text on images for visually impaired users.
  • High Contrast and optimized font sizes for readability.

Content Page

Technical Description

The Content page is implemented using HTML,CSS and JS to provide a structured and visually appealing interface.

HTML elements

  • (header & nav) - Houses the logo, navigation links, and a hamburger menu for mobile-friendly navigation.
  • (section class="News") - Contains news articles.
  • (div class="program-card") - Displays each news item with an image, title, and “Read” button linking to full articles.

CSS Techniques

  • Global Styling - Consistent spacing, typography, and box-sizing.
  • Cards Design - (.program-card) applies box-shadow, border-radius, and flexbox for a clean, structured layout.
  • Dropdown Menu - (.dropdown-content) is initially hidden and appears on interaction.

Accessibility

These techniques below improve the accessibility of the news content.

  • Alt Text for Images - Ensures visually impaired users get meaningful descriptions.
  • Semantic HTML - Uses (header), (nav), and (section) for better screen reader support.
  • High Contrast & Readability - Adequate color contrast, readable font sizes, and clear button styles.

Challenges and Lessons Learned

During the implementation process, several challenges were encountered, requiring troubleshooting and problem-solving skills.

  • The dropdown menu for selecting years initially did not function as expected, remaining hidden even when clicked. This issue was traced to the JavaScript function controlling the dropdown. To resolve this, the toggleDropdown() function was updated to correctly add and remove the .show class, ensuring the menu displayed when needed.
  • The news section cards were not aligning properly, especially on smaller screens. This was due to improper CSS layout settings. The issue was resolved by modifying the .program-container using display: flex with justify-content: center; and align-items: stretch;. Additionally, gap: 30px; was added to create proper spacing between the cards.
  • The images inside the program cards did not fit correctly, often appearing stretched or distorted. This was due to inconsistent aspect ratios. The issue was resolved by adding the object-fit: cover; property to the image styling, ensuring that all images maintained their proportions while properly filling their designated space.

    Lessons Learned

    • Debugging JavaScript requires careful inspection using browser developer tools.
    • Proper CSS structuring, including flexbox and grid, significantly improves layout consistency.
